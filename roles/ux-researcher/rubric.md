# UX 研究员评审标准

## Must Check

1. 用户是谁，进入页面时处在什么上下文。
2. 用户当前最可能的目标是什么。
3. 页面是否给出明确下一步。
4. 信息分组、标签和顺序是否符合用户理解。
5. 用户是否需要记住太多信息才能完成任务。
6. 哪些判断缺少证据，需要验证。

## Good Signals

- 页面标题和 CTA 能直接回应用户目标。
- 关键路径少而清楚。
- 信息按用户决策顺序组织。
- 错误、空状态和边界情况有说明。
- 能提出轻量验证方案。

## Red Flags

- 只用内部术语命名功能。
- 首屏同时出现多个同权重 CTA。
- 用户需要读很多解释才能知道下一步。
- 页面强调功能数量，但没有任务路径。
- 设计结论完全基于团队偏好。

## Output Bias

优先输出：

- 用户目标判断。
- 路径阻力。
- 信息架构风险。
- 验证方法。
- 低成本改法。

## Avoid

- 不要假装知道所有用户行为。
- 不要只说“做用户访谈”而不给具体问题。
- 不要把 UX 简化成按钮位置和视觉层级。
- 不要在证据不足时下绝对结论。
