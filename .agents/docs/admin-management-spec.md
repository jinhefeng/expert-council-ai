# 功能说明书：后台管理与前端体验优化

## 1. 业务目标
提升主会议页面的操作体验，并抽离底层配置项（系统专家与大模型配置），将它们集中到专属的后台管理页面中，使主页面的核心聚焦于圆桌会议本身。

## 2. 需求拆解与具体方案

### 2.1 主页面前端体验优化 (`/src/app/page.tsx`)
- **专家席位排序优化**：在左侧“专家席位”列表中，当前会议已勾选（激活）的参会专家将被优先置顶显示，未参会的沉底，方便用户快速查看和调整当前的参会阵容。
- **新建按钮移位**：将底部的“+ 新建组织智能体”按钮移至“专家席位”区块的标题栏右侧，与“会议空间”的“+”按钮布局保持对称和一致性，提升视觉整洁度。
- **配置面板精简**：随着大模型配置管理被移至后台，主页右侧的“控制面板”将不再负责新建和编辑引擎。如果需要，只保留一个当前所选引擎的展示（或快捷切换），以减少主界面的冗余信息。

### 2.2 新增后台管理页面 (`/src/app/admin/page.tsx`)
- **页面入口**：在主界面的页头（Header）增加一个“后台管理”的跳转入口。
- **功能一：系统内置专家与组织自定义专家管理**
  - **展示层**：列表展示所有的系统内置专家以及自定义专家。
  - **交互层**：支持编辑系统内置专家（例如修改系统专家的判断视角、性格、Prompt 等）和自定义专家。
  - **存储机制**：修改系统内置专家时，原有的 `src/lib/experts.ts` 作为种子数据（Seed Data）。修改后的内容将保存在 `localStorage` 中的 `system-experts-override` 字段中，并在运行时与种子数据进行 Merge（合并）。
- **功能二：大模型配置管理 (LLM Engine Management)**
  - 将原先在主页的“组织大模型配置”功能完全迁移至该页面。
  - 支持“新建自定义大模型”、“编辑”、“删除”、“设为默认激活”、“导出配置”和“导入配置”等功能。

## 3. 影响范围 (Impact Analysis)
1. **数据层 (`src/lib/storage-service.ts`)**：
   - 新增 `getSystemExpertOverrides` 和 `saveSystemExpertOverrides` 方法。
2. **逻辑层 (`src/lib/model-router.ts` 等)**：
   - 依赖 `defaultExperts` 的逻辑（包括分配发言人时的过滤逻辑）需要适配新的合并后的专家列表（Merged Experts），不再单纯读取静态常量。
3. **UI 层 (`src/app/page.tsx` & `/src/app/admin/page.tsx`)**：
   - 主页的代码量将得到精简，大模型的编辑弹窗和配置列表代码将迁移至 `admin` 页面。

## 4. 边缘案例处理
- **恢复默认**：针对系统内置专家，提供“恢复默认配置”的操作按钮，以清除针对该专家的 override 记录。
- **导入冲突**：在大模型配置导入时，若 ID 已存在则进行覆盖。
