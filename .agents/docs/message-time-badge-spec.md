# 功能规格与交互说明书：聊天气泡时间与耗时展示

本项目为专家发言和主持人总结气泡引入了两个精细的时间度量展示，用以提供全方位的会议记录追溯体验。

## 1. 业务逻辑与规格定义

1. **绝对出现时刻**：
   - 记录消息产生的绝对时间戳，精确到秒。
   - 格式统一化为 `HH:mm:ss`（例如 `22:30:15`）。
2. **流式生成耗时**：
   - 专属于大模型生成或总结的信息（如专家发言和主持人总结）。
   - 统计大模型从发起 API 请求、开始返回首字（TTFB）、直到流式吐字完全结束的实际流转毫秒数。
   - 在前端换算并四舍五入为精度至 1 位小数 of 1 秒数（例如 `4.2s`）。
   - 保存在消息实体（`ChatMessage`）的 `duration` 字段中。

## 2. 视觉与交互设计 (UI/UX Spec)

### 2.1 对齐与布局
我们秉持**顶级设计师**的高标准规范，让包含“名字”与“时间”的元数据行在视觉上与气泡边界严格垂直对齐：
- **常态化展示**：信息常驻于气泡上方，无需 hover 触发显示。
- **两端对齐**：使用 `justify-content: space-between`。
- **左上角与右上角精准吸附**：
  - **用户侧（居右）**：使用 `flex-direction: row-reverse` 容器，让名字居右，时间与耗时居左。从而使得时间绝对对齐黑色气泡的**左上角边缘**。
  - **专家与主持侧（居左）**：名字居左，时间与耗时居右。从而使得时间绝对对齐气泡的**右上角边缘**。
- **气泡同宽内聚**：将元数据行与气泡内容（`message-content`）封装在同一个 `width: fit-content` 纵向 Flex 容器中。这使得当气泡因内容短而收缩时，时间信息会自动精准地向内靠拢，严密贴近气泡边界，解决了拉伸过宽的排版缺陷。

### 2.2 图标与字体样式
- **扁平科技感图标**：
  - 不使用彩色的 emoji。
  - **出现时刻**：使用极简扁平的细线圆环时钟 SVG 图标 (`🕒` 效果)。
  - **发言耗时**：使用极简扁平的单色闪电 SVG 图标 (`⚡` 效果)。
- **字色与字号**：字号保持 `11px`，字色为内敛的 `var(--muted)`，闪电与耗时数值辅以 `var(--amber)` 高亮展示，完美承袭发言人名字样式。
- **防抖 Tabular Nums**：数字使用 `tabular-nums` 等宽字体，避免每次生成时间时因数字宽度不同产生视觉抖动。

## 3. 技术实现细节

### 3.1 扩展的类型定义
在 `src/lib/types.ts` 的 `ChatMessage` 中增加：
```typescript
export type ChatMessage = TenantScoped & {
  ...
  duration?: number; // 发言/总结生成耗时 (秒)
};
```

### 3.2 耗时记录点
在 `src/app/page.tsx` 中如下场景采集并保存 `duration`：
- **初始总结**：`requestSynthesis` 执行前后计算并绑定。
- **一轮专家发言**：`requestExpertTurn` 发生 try/catch 前后计算并绑定。
- **流内主持人总结**：`requestSynthesis` 执行前后计算并绑定并异步合并到 `modMessage`。

### 3.3 编译与渲染防抖
- 在 `ChatMessageCard.tsx` 中重构 `areEqual` 方法，将 `prevProps.message.duration === nextProps.message.duration` 列入 React.memo 对比链，保障精确的局部 Reflow。
