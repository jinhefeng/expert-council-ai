# Design Council AI 产品使用说明书

欢迎使用 Design Council AI。本项目旨在提供一个**本地运行的 AI 专家讨论平台**。通过模拟现实世界的多领域专家圆桌会议，帮助你进行产品功能梳理、架构设计评审及体验可行性评估。

## 1. 初始化设置

### 配置模型
由于 AI 会议强依赖于大型语言模型，在使用前请确保你已经在本地根目录 `.env.local` 文件中配置了相应的 `DASHSCOPE_API_KEY`（目前默认主推通义千问）或者其他兼容接口。如果没有配置，前端应用会发出警告，且可能仅能以 Mock 模式运行。

### 本地启动
在项目根目录运行 `pnpm dev` 并在浏览器中访问 `http://localhost:3000` 即可看到主界面。

## 2. 理解专家角色 (Roles)

本系统在底层及 `roles/` 目录下存储了一批预设的专家角色。每位专家都有以下属性：
- **名称与头衔**：例如“UX 研究员”、“前端架构师”。
- **脾气/气质**：如“逻辑严密”、“发散性强”。
- **核心关注点**：如“系统性能”、“用户心理”。

**在界面上：**
你可以直接勾选想要邀请加入本轮圆桌会议的专家。不同的议题，你应当邀请不同背景的专家来获得多角度视角的综合方案。

## 3. 发起讨论流程

1. **选择会议主持模式**
   - 你可以选择不同的“主持人策略”（如“温和折中”、“激进创新”或“严格推进”）。主持人的不同会影响最终输出的**综合结论（Synthesis）**的倾斜度。
2. **设定议题**
   - 在输入框中详细写下你要讨论的问题。
3. **上传资料附件 (可选)**
   - 如果你有产品需求文档（Word / Markdown / PDF）或 UI 设计草图（图片），可以通过界面直接选择或拖拽上传。所有的文件仅在你的本地解析并转化为文本或特定的 Token 给到大模型。
4. **启动圆桌会议**
   - 点击“开始讨论”后，大模型将接管整个会议进程。

## 4. 读懂输出结果

整个会议将呈现为一个**多步骤的自动流**，包含：

- **智能指派下一个专家**：系统会自动判断谁最适合首先对议题进行发言。
- **专家流式发言（可能带长考）**：具有深度推理能力（Reasoning）的模型可能会输出包含思考过程的文字（在折叠面板 `<think>` 标签内展示）。
- **四象限卡片输出**：每个专家发言后，不仅有直接的对话内容，通常还会附带一个结构化的 JSON 总结，包含：
  - `Stance (立场)`
  - `Concern (风险)`
  - `Recommendation (建议)`
  - `Tradeoff (取舍)`
- **主持人终结**：经过数个回合（Round）后，主持人将收集各方观点，出具一份全场总综合与下一步行动指南（Action Items）。

## 5. 常见问题 (FAQ)

- **为什么专家发言突然中断？**
  这是由于模型在单次生成时超过了最大的 Token 限制。如果你发现推理模型总是因为“想得太多”被截断，可以考虑减少一次性参加的专家数量或者联系开发人员扩大 API 最大输出长度设置。
- **我能自己修改专家的性格吗？**
  是的。目前你可以通过进入代码目录的 `roles/` 文件夹修改对应专家的 `.md` 知识库和 `src/lib/experts.ts` 的配置。未来应用可能会提供可视化界面直接新建本地专家。
