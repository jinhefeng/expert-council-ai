# 功能说明书：主界面布局与视觉优化

## 1. 业务目标
基于目前的 UI 截图反馈，优化整个页面的组件排版、对齐方式以及视觉重心，解决由于新增元素导致的布局错位和割裂感。

## 2. 存在的不合理点与改进方案

### 2.1 顶部 Header 区域
**问题**：右上角的“后台管理”按钮呈巨大的深色块状，且在 Flex 布局中换行或与前方的状态标签（Status Chips）未保持居中对齐，显得非常突兀。
**改进**：
- 调整 `.status-group` 的对齐方式，确保所有内容单行垂直居中（`align-items: center`）。
- 将“后台管理”按钮样式从厚重的 `secondary-button` 改为带浅色边框的扁平按钮（如 `ghost-button`），或者调整其 padding 和 height，使其和左侧的 chip 高度一致。

### 2.2 专家席位面板 (专家席位 Panel)
**问题**：标题栏中的“+ 新建”按钮样式过于笨重，且与右侧的收起箭头（›）挤在一起。它与左侧“会议空间”面板右上角那种轻量的 `+` 按钮风格不一致。
**改进**：
- 将“+ 新建”按钮修改为类似“会议空间”的轻量化图标按钮或幽灵按钮。
- 调整外层 flex 布局，确保 Title 居左，操作按钮居右且间距合理。

### 2.3 底部输入区 (Composer)
**问题**：底部聊天输入框区域（`.composer-row`）的左侧 `+` 附件按钮与右侧的文本输入框没有完美垂直居中对齐。输入框内的“召开圆桌”按钮颜色和体积较为抢眼。
**改进**：
- 优化 `.composer-row` 的 `align-items: center` 或 `align-items: flex-end`，并调整文本域的基础高度，使得左右两侧控件视觉平衡。

### 2.4 右侧控制面板
**问题**：“前往后台管理大模型 ↗”仅仅是一个裸露的红色文字链接，显得过于单薄且缺乏可点击感。
**改进**：
- 将其包裹在类似 `secondary-button` 或带边框的块级样式中，提升点击区域和视觉规范性。

## 3. 影响范围
- 主要涉及 `src/app/page.tsx` 中的 JSX class 命名与内联样式调整。
- 可能会在 `src/app/globals.css` 中微调对应 class 的 padding 或 margin。
- 仅为前端表现层修改，不影响数据与核心逻辑。
