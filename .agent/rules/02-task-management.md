# 全局任务管理准则 (Task Management Skill)

作为 Antigravity 助手，在本项目中，你被赋予了**全局任务管理者**的职能。你的工作不仅是完成单次任务，还需要跨会话协助用户维护全局的项目开发进度。

## 核心任务池
所有的长期任务、功能规划、迭代进度，都维护在这个核心文件中：
`[Project Tasks](file:///Users/jinhefeng/Dev/design-council-ai/.agent/docs/project-tasks.md)`

## 触发机制与职责 (When & What)

1. **会话开启时（感知进度）**：
   - 每次接到涉及到开发新功能或修复复杂缺陷的任务前，**主动读取**上述 `project-tasks.md` 文件，了解当前进度和是否有被搁置的任务。
2. **制定计划时（拆解任务）**：
   - 当用户提出新需求，或者当你生成了一份《功能说明书》后，必须**主动更新** `project-tasks.md`。
   - 使用 `replace_file_content` 或类似工具，将宏观需求拆解为具体的小项，添加到 `TODO` 列表中。
3. **完成任务时（更新状态）**：
   - 当某项工作经过验证完成，或者取得用户的 `LGTM` 后，在回复用户前，**主动更新** `project-tasks.md`。
   - 将对应的任务项从 `[ ]` 改为 `[x]`，并视需要移动到 `✅ 已完成 (Done)` 区域。
4. **管理指令（被动触发）**：
   - 当用户要求“检查当前任务”、“列出所有未完成项”、“帮我把某项标为完成”等情况时，直接读取该文件并予以反馈和执行。

## 严格工作流 (Strict Workflow)
- **不可遗忘**：每次更新代码状态后，必须确保 `project-tasks.md` 与实际开发进度保持一致。这是实现你“长时记忆（Long-term memory）”和“任务延续性”的核心手段。
- **格式要求**：保持 Markdown 格式中的 `[ ]` 和 `[x]` 标记正确。避免重复添加相同任务。

> 你的行为准则优先级：用户明确指定的操作 > 本规则维护 > 单次临时分析。
